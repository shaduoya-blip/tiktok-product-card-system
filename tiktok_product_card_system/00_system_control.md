# 00 System Control

## 文件用途
定义系统处理 TikTok Shop US 商品卡规划任务的工作顺序和基础边界。

## 系统工作顺序
1. 读取用户产品信息
2. 如果有竞品商品卡，先拆解竞品
3. 如果有竞品图片，分析主图/详情图内容、文字和排版
4. 提取候选卖点
5. 给候选卖点评分
6. 确定最大卖点
7. 规划标题、搜索关键词、主图、详情图、描述、5点亮点
8. 为每张主图/详情图单独输出 ChatGPT 生图提示词
9. 做质量检查
10. 标记后续可补充信息

## 基础规则
- 用户上传的竞品商品卡内容默认可参考、可拆解、可转化。
- 不建立阻断式风险审查系统。
- 不主动编造用户和竞品都没有的信息。
- 缺信息时继续输出，但标记“可补充信息”。

## 竞品资料读取控制规则（2026-07-08补充）
- 读取竞品资料前必须先递归核验 Git 状态、当前 HEAD、`raw_inputs/竞品资料/` 实际目录和 Git 跟踪清单。
- 不得因根目录存在 `.gitkeep` 就判断竞品资料为空；必须进入每个竞品文件夹、主图、详情图、SKU/产品参考图和 zip 包清单。
- 中文路径必须按 UTF-8 处理并加引号；不得重命名、移动、覆盖用户上传的原始竞品文件。
- 图片无法打开时必须记录具体路径并列入视觉复核清单，不得根据文件名编造图片内容。

## 同一产品多卖家聚合控制规则（2026-07-08补充）
- 竞品资料目录下如存在 `产品分组说明.txt`，必须优先按该文件建立产品组；只有缺少分组说明时，才允许根据文件夹名称自动推断分组，并在输出中标记“自动推断分组”。
- 支持两种目录方式：产品组嵌套目录（`竞品资料/产品组/卖家竞品/`）和平铺目录（`竞品资料/卖家竞品1/`、`竞品资料/卖家竞品2/`、`产品分组说明.txt`）。
- 同一产品的多个卖家商品卡必须先独立拆解并保留来源，再做产品组级综合；不得把规格、参数、配件或卖点简单相加。
- 多卖家内容冲突时，必须分别保留来源，不得强行选择、平均或合并成无来源结论。
