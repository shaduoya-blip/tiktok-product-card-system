## 1. 系统任务

系统负责为 TikTok Shop US 商品卡生成可执行规划，核心任务包括：

- 理解用户当前提供的真实产品信息、SKU、规格、价格、包装内容和产品参考图。
- 拆解竞品商品卡、竞品图片和同款产品多个卖家的商品卡样本。
- 聚合同一产品多个卖家样本，提取共同卖点、独有卖点、规格差异、信息冲突和机会点。
- 提取候选卖点，评分并确定最大卖点、次级卖点、标题优先卖点和首图优先卖点。
- 规划 TikTok Shop US 标题、搜索关键词、主图 6–8 张、详情图、商品描述和 5 点商品亮点。
- 为每一张主图和每一张详情图单独输出适合 ChatGPT 生图的英文提示词。
- 做字符数、信息来源、美国本土表达、图片顺序、规格一致性和生图锁定项检查。

## 2. 信息优先级

执行时按以下优先级判断信息可信度和采用顺序：

1. 用户当前提供的我方真实产品信息。
2. 用户当前上传的产品参考图。
3. 用户当前指定的产品规格、SKU、价格、折扣和包装内容。
4. 仓库中的产品档案。
5. 对应产品组的竞品分析、竞品图片分析和产品分组说明。
6. 仓库通用规则、专项策略文件和知识卡。
7. 历史对话信息。

历史对话不能覆盖当前用户输入，旧任务分支或旧对话规则不能替代最新 `main` 运行包。

## 3. 标准执行流程

1. 读取用户当前产品信息，确认产品类型、规格、SKU、数量、配件、包装和可验证参数。
2. 读取产品参考图，锁定结构、颜色、比例、数量、配件、材质和关键识别点。
3. 读取相关竞品产品组；如有 `raw_inputs/竞品资料/产品分组说明.txt`，优先按该文件建立产品组。
4. 独立拆解各竞品卖家的标题、SKU、商品信息、主图、详情图、描述、亮点和图片排版。
5. 聚合同一产品多个卖家样本，区分共同卖点、独有卖点、表达差异、规格差异、信息冲突、共同图片逻辑和机会点。
6. 建立候选卖点池，保留来源，不把多个竞品优点简单累加给我方商品。
7. 按评分维度给候选卖点评分。
8. 确定最大卖点、次级卖点、标题优先卖点、首图优先卖点和详情图补充卖点。
9. 规划标题和搜索关键词，使平台能理解商品，同时自然覆盖核心搜索意图。
10. 规划主图和详情图，使图片承接点击、理解、证明、信任和转化。
11. 输出商品描述和 5 点商品亮点，先写用户收益，再写产品特征。
12. 为每张主图和详情图单独输出 ChatGPT 生图提示词，不合并成一条大提示词。
13. 执行质量检查，列出可补充信息和最终优化建议。

## 4. 输入资料识别

需要识别并分别处理以下资料：

| 输入类型 | 处理规则 |
|---|---|
| 用户产品资料 | 默认可用，但只写成用户已提供或可由图片确认的事实。 |
| SKU / 规格 / 颜色 / 尺寸 | 必须与用户当前商品一致，不得混用竞品不同版本。 |
| 价格与折扣 | 仅在用户提供时使用，不凭竞品价格推断。 |
| 包装内容 | 只写可确认内容；不把不同卖家的配件合并为我方包装。 |
| 产品参考图 | 生图时优先作为身份锁；锁定结构、颜色、比例、数量和配件。 |
| 中文竞品资料文件夹 | 支持中文路径、中文图片名、主图1/详情图1等通用命名。 |
| 主图1、主图2…… | 必须看图或读取已有图片分析后判断作用，不能只根据文件名推测。 |
| 详情图1、详情图2…… | 记录文字、信息层级、买家问题、对应卖点和可借鉴结构。 |
| zip 包 | 可临时解压读取，但不得改动原始 zip。 |

缺少信息时继续规划，并在“可补充信息”中列出需要用户确认的内容。

## 5. 竞品产品组聚合规则

### 5.1 产品组建立

- 竞品资料目录下如有 `产品分组说明.txt`，必须优先按该文件建立产品组。
- 支持产品组嵌套目录：`竞品资料/产品组/卖家竞品/`。
- 支持平铺目录：`竞品资料/卖家竞品1/`、`竞品资料/卖家竞品2/`，并用分组说明绑定。
- 缺少分组说明时，才可根据文件夹名自动推断，并在输出中标记“自动推断分组”。

### 5.2 独立拆解

同一产品的多个卖家样本必须先分别拆解：

- 标题结构、首位关键词和标题优先卖点。
- SKU、颜色、尺寸、数量、配件和版本信息。
- 商品详细信息、描述结构和 5 点亮点。
- 主图顺序、详情图顺序、图片文字、转化任务和排版逻辑。
- 候选卖点、最大卖点、首图优先卖点和详情图补充卖点。
- 美国本土表达方式和可复用短文案模式。

### 5.3 产品组综合

产品组层级只综合可复用规律：

- 共同卖点：多个卖家都强调或图片多次证明的稳定卖点。
- 独有卖点：仅某个卖家出现但来源清楚的卖点，保留来源。
- 表达差异：同一卖点在不同卖家中的标题、图片、描述表达差异。
- 规格差异：尺寸、颜色、配件、数量、材质、版本等差异，不得混为一谈。
- 信息冲突：用“卖家A说法 / 卖家B说法 / 待我方确认”呈现。
- 共同标题结构：核心词、最大卖点、场景词、套装词、人群词的常见顺序。
- 共同关键词方向：主词、场景词、人群词、用途词、痛点词、礼品词。
- 共同图片逻辑：首图效果展示、功能证明、尺寸、配件、步骤、电池/充电、场景收口等。
- 未充分表达的机会点：竞品没有讲清楚但我方可用图片或文案补强的转化点。

不得简单拼接全部卖点，不得把不同规格、不同配件或不同参数合成无来源事实。

## 6. 最大卖点提取规则

### 6.1 候选卖点来源

候选卖点来自：

- 用户当前产品信息。
- 产品参考图可确认的结构、颜色、数量、配件和使用方式。
- 竞品标题、首图、详情图、描述和亮点中反复出现的卖点。
- 竞品图片中可被一眼证明的效果、场景、配件、尺寸、步骤或信任信息。
- 产品组共同卖点和有来源的独有卖点。

### 6.2 评分维度

每个候选卖点按 1 / 3 / 5 分评估：

| 维度 | 判断问题 | 高分标准 |
|---|---|---|
| 买家欲望强度 | 买家是否真的在意？ | 能直接触发购买理由。 |
| 视觉证明强度 | 是否一眼看懂？ | 可通过首图或详情图直接证明。 |
| TikTok停留吸引力 | 是否能快速抓住注意？ | 适合动态、效果、反差或强场景展示。 |
| 搜索相关性 | 是否能带出真实搜索词？ | 能自然延展主词、用途词、场景词。 |
| 竞品差异化 | 是否比竞品更突出？ | 能形成我方角度或表达优势。 |
| 转化推动力 | 是否推动下单？ | 能降低疑虑或提高感知价值。 |
| 主图适配度 | 是否适合前3张主图？ | 可成为点击图或核心证明图。 |
| 标题前置价值 | 是否值得放在标题前段？ | 是买家和平台都能快速识别的价值。 |
| 美国本土表达适配度 | 是否能自然写成美国电商语言？ | 英文短语自然、不像直译。 |
| 来源支持度 | 是否有来源支撑？ | 用户输入、多卖家共同强调或图片清楚证明。 |

来源支持度分级：两个及以上卖家共同强调为较高；一个卖家强调且图片清楚证明为中高；只有一个卖家文字提到为中等；模型自行推断为低，只能作为建议。

### 6.3 最大卖点判断

- 最大卖点不能只看标题；必须交叉验证标题首位关键词、首图视觉焦点、多张图片重复表达、详情图证明链和一眼可视化程度。
- 儿童套装类优先提取“完成后的可玩结果”，再提取教育、配件数量、送礼和说明书。
- 电动玩具类优先提取“自动 / 一键 / 连续动作 + 可视效果”，再提取射程、灯效、电池、尺寸。
- 泡泡、水枪等效果型产品，首图最大卖点必须能被效果图证明，如泡泡量、射击水束、灯效、动态轨迹或完成场景。

### 6.4 最大卖点贯穿

- 标题：最大卖点靠前，但保持自然，不牺牲核心产品词。
- 首图：用最直观画面表现最大卖点，必要时用 2–5 个英文词短文案。
- 详情图：建立证明链，解释为什么该卖点可信、如何使用、包含什么或适合什么场景。
- 商品亮点：前 1–2 条覆盖最大卖点和首图最大卖点。
- 生图提示词：把最大卖点转成具体画面动作、视觉效果、场景或对比，而不是抽象口号。

## 7. 标题规则

- 标题最多 255 个字符。
- 标题必须包含核心产品词，帮助平台识别商品类别。
- 最大卖点应尽量前置，但不要破坏产品识别。
- 自然覆盖使用场景、人群/场合、套装/颜色/规格等高价值信息。
- 使用美国本土电商表达，避免中文直译腔和跨境模板腔。
- 不做无意义堆词，不塞入与商品无关的热门词。
- 不复制竞品完整标题；只吸收关键词顺序和结构。
- 未经确认的数字、认证、材质、年龄段、兼容性或性能参数不得写成事实。

推荐结构：

`核心产品词 + 最大卖点/核心功能 + 规格/套装 + 使用场景 + 人群/礼品/场合`

可按类目调整顺序：

- 效果型玩具：效果或动作可以靠前。
- 配件套装：套装数量和包含内容可以靠前。
- 礼品型商品：产品词之后补充 gift / party / birthday / outdoor 等场景词。

## 8. 搜索关键词规则

- 搜索关键词最多 250 个字符。
- 关键词用于帮助平台理解商品，不是给买家阅读的句子。
- 覆盖主词、同义词、场景词、人群词、用途词、痛点词、礼品词、季节词和最大卖点延伸词。
- 优先选择与产品真实相关、买家可能搜索、能被商品卡内容承接的词。
- 不堆无关词，不加入竞品品牌词，不加入无法证明的功能词。
- 标题、关键词、图片、描述和亮点中的核心表达要一致，避免搜索逻辑和转化逻辑互相破坏。

关键词分组建议：

| 类型 | 示例方向 |
|---|---|
| 主词 | product type, category name |
| 场景词 | outdoor, pool, party, backyard, classroom |
| 人群词 | kids, toddlers, boys, girls, parents |
| 用途词 | summer play, birthday gift, party favor |
| 痛点词 | easy setup, no mess, portable, reusable |
| 礼品词 | gift for kids, birthday gift, holiday gift |
| 季节词 | summer, Easter, Halloween, Christmas，按产品适配使用 |
| 卖点延伸词 | one-button, rechargeable, kit includes, LED effects |

## 9. 主图规划规则

默认规划 6–8 张主图。顺序可按产品类别调整，但必须符合点击、理解、证明、信任和转化链路。

1. 首图 / 产品识别和最大卖点：展示产品和最大可视化结果，用于点击；效果型产品优先展示动作、泡泡、水束、灯效、完成场景或强场景结果。
2. 包装内容 / 套装价值：展示 kit includes / full set，避免买家误解数量和配件。
3. 核心使用场景：展示真实使用环境和目标人群，帮助买家想象拥有后的体验。
4. 最大卖点视觉证明：用按钮特写、箭头轨迹、光效、对比、四宫格或数据图表证明核心功能。
5. 尺寸比例：用尺寸线、手持比例、儿童/成人比例或场景比例降低理解阻力。
6. 生活方式 / 礼品 / 场合：把功能转成 party、outdoor、family fun、birthday gift 等购买理由。
7. 对比或疑虑解决，可选：解决安装、补水、充电、清洁、收纳、耐用、使用门槛等疑虑。
8. 细节特写，可选：展示材质、按钮、接口、配件细节或安全边角。

调整顺序的情况：

- 如果产品靠效果吸引点击，效果图优先于参数图。
- 如果产品靠套装价值转化，包装内容可提前到第 2 张。
- 如果买家疑虑强，尺寸、步骤、充电或安装图应提前。
- 如果图中文字多，拆成多张；移动端单图只承担一个主要任务。
- 主图英文文字优先 2–5 个词，副标题不超过一行，避免遮挡产品关键结构。

## 10. 详情图规划规则

详情图用于承接首图兴趣，并逐步解决买家的核心问题：

- 一张详情图解决一个主要问题。
- 移动端优先，信息层级清楚，标题短，参数真实。
- 详情首图承接最大卖点，但用更明确标题解释买家为什么需要它。
- 后续按买家疑问排序：怎么操作、性能如何、包含什么、怎么安装/补水/充电、尺寸多大、适合什么场景。
- 可包含卖点拆解、使用步骤、包装内容、尺寸、使用场景、对比、FAQ、礼品/场合。
- 汇总图信息过密时拆成多张单点图。
- 结尾用生活方式或使用场景图，把功能转成派对、户外、亲子、礼物等购买理由。

常用详情图结构：

1. 最大卖点解释图：明确核心利益和使用结果。
2. 使用步骤图：3–4 步，箭头或局部特写。
3. 功能证明图：按钮、轨迹、灯效、射程、泡泡量、材质或稳定性。
4. 包装内容图：配件平铺、数量标注、kit includes。
5. 尺寸和规格图：真实数字、in/ft 单位、清晰尺寸线。
6. FAQ / 疑虑解决图：充电、安装、补水、适配、清洁、收纳等。
7. 场景收口图：party、outdoor、gift、family fun。

## 11. 商品描述规则

商品描述应自然、简洁、美国本土化，不机械复制竞品：

- 第一段：说明这是什么产品，以及最大可视化卖点或核心使用结果。
- 第二段：说明使用场景和体验，把功能转成买家收益。
- 第三段：说明包装、规格、活动或礼品用途；只写用户或图片可确认的信息。
- 每个功能点尽量对应图片可证明内容，如配件、按钮、灯效、射程、泡泡量、尺寸、电池、说明书。
- 未经我方验证的竞品数字不得写入描述；可改为真实参数或温和表达，如 `designed for longer play`。
- 避免夸张绝对化、医疗化、安全保证、认证承诺或无法证明的性能宣称。

## 12. 5点商品亮点规则

5 点亮点总字符最多 1500。结构建议：

1. 最大卖点：标题短语 + 一句用户收益。
2. 易用性：操作、安装、充电、补水、收纳或清洁等低门槛信息。
3. 使用场景：party、outdoor、pool、backyard、family fun、classroom 等按产品适配。
4. 包装和价值：kit includes、full set、配件、数量或可确认套装价值。
5. 礼品 / 场合 / 购买理由：birthday gift、holiday gift、summer play、party favor 等。

写法要求：

- 先用户收益，后产品特征。
- 自然覆盖关键词，不堆词。
- 前 2 条必须覆盖标题最大卖点和首图最大卖点。
- 每条亮点尽量对应一张主图或详情图，避免图片无法证明的卖点。
- 常用短标题可参考：`One-Button Fun`、`Long-Range Spray`、`Kit Includes`、`Party-Ready Bubbles`、`Easy Setup`。

## 13. 美国本土电商语言规则

- 使用美国买家熟悉的短句、短语和利益表达，避免中文直译。
- 移动端图片文字优先 2–5 个英文词；副标题不超过一行。
- 参数用真实数字和美国常用单位，如 in、ft、oz。
- 语气随类目调整：玩具可活泼，家居可实用，礼品可温暖，电子配件可清晰直接。
- 不机械套用固定例句，要根据产品和场景自然改写。

推荐表达模式：

- `One-Button Blast`
- `Long-Range Spray`
- `Bright LED Effects`
- `Rechargeable Battery`
- `Kit Includes`
- `Full Set Included`
- `Party-Ready Bubbles`
- `Easy Setup`
- `Product Details`

避免表达：

- 生硬直译：`remote firing range`、`simulate the light effect of a fire jet`。
- 夸张模板：`best ever`、`perfect for everyone`、`must-have for all`。
- 难以证明的承诺：`100% safe`、`guaranteed`、`professional grade`，除非用户提供证据。

## 14. TikTok Shop 流量和转化逻辑

- 标题和搜索关键词帮助平台理解商品、匹配搜索和推荐场景。
- 图片、描述和亮点负责承接点击后的理解、信任和下单。
- 最大卖点必须贯穿标题、首图、详情图、描述、亮点和生图提示词。
- 搜索逻辑与转化逻辑不能互相破坏：不能为了关键词写入图片和描述无法承接的卖点。
- 各字段表达要一致：同一个功能不要在标题、图片、描述中写成互相冲突的版本。
- 低客单或冲动购买商品要降低理解成本，让买家先看懂结果，再看细节。

## 15. 购物心理学应用

精简使用以下心理原则：

- 低客单冲动购买：首图和标题要快速给出“为什么现在买”。
- 视觉先于文字：能用画面证明的，不用长文解释。
- 感知价值：通过套装、配件、尺寸、场景和礼品用途提高“值”的感受。
- 降低理解阻力：一图一任务，步骤清楚，参数真实，文字短。
- 场景化购买：把功能转成 backyard、pool、party、family fun 等具体使用画面。
- 礼品属性：对儿童、季节、派对、生日等商品强调 gift-ready 理由。
- 家长购买心理：强调易用、可玩结果、亲子互动、收纳、配件清楚、信息可信。
- 问题—解决方案—收益结构：先识别买家疑虑，再用产品信息解决，最后转成利益。

## 16. 竞品图片分析规则

分析竞品图片时，必须基于实际图片或已有图片分析记录，不得只根据文件名推测。

每张主图记录：

- 画面内容。
- 图片文字、主标题、副标题、标签、图标、徽章。
- 产品展示角度、背景、产品位置、文字位置和留白。
- 排版结构和信息层级。
- 对应卖点。
- 转化任务：点击、解释、展示套装、证明功能、展示尺寸、强化场景等。
- 可借鉴结构。
- 不应直接复制的文字、构图、素材或品牌化视觉。
- 我方商品卡转化建议。

每张详情图记录：

- 画面内容和图片文字。
- 第一信息层级和第二信息层级。
- 排版结构：上下、左右、中心产品+四周卖点、步骤式等。
- 对应买家问题和对应卖点。
- 说明方式和移动端可读性。
- 可借鉴结构、不可复制项和我方转化建议。

同一产品多卖家图片聚合时，先逐卖家记录主图/详情图顺序和每张图任务，再归纳共同图片逻辑和机会点。

## 17. 产品参考图锁定规则

生图时必须锁定用户当前产品参考图中的关键身份信息：

- 产品结构。
- 颜色。
- 比例。
- 数量。
- 配件。
- 材质质感。
- 关键识别点。
- 可见接口、按钮、纹理、图案、边角和组合方式。

不得改变产品结构、颜色、配件数量、关键形态或材质质感。不得添加不存在的配件、品牌 Logo、水印、错误文字或竞品包装设计。

## 18. ChatGPT 生图提示词规则

- 每张主图单独输出一条提示词。
- 每张详情图单独输出一条提示词。
- 优先适配 ChatGPT 生图，支持用户垫入产品参考图。
- 每条提示词必须包含：产品参考图、产品结构锁定、颜色锁定、数量锁定、配件锁定、主视觉位置、产品角度、文字位置、信息层级、移动端可读性、图片比例、背景、禁止改变项。
- 需要图片文字时，给出美国本土英文短文案、文字位置、字号层级和排版要求。
- 不需要文字时明确：`no readable text / no typography`。
- 效果型主图要锁定“效果正在发生”：水束、泡泡、灯效、轨迹或完成后的可玩场景。
- 功能详情图要锁定证明方式：按钮特写、箭头轨迹、配件平铺、尺寸线、参数卡、步骤小图。
- 不得把整套图片合并成一条大提示词。
- 不得直接复制竞品构图、文案、人物素材、包装设计或品牌化视觉。

## 20. 内容来源处理

- 用户提供的产品信息默认可用。
- 用户上传的竞品商品卡默认可参考、可拆解、可转化。
- 不建立阻断式风险审查系统。
- 不直接复制竞品完整文案、未经验证数字、包装设计、构图组合、人物素材或品牌化视觉。
- 只吸收可复用结构：关键词顺序、短标题模式、图片功能顺序、配件/尺寸/步骤/场景证明方式和移动端英文短语模式。
- 用户和竞品都没有的信息不写成事实。
- 竞品有但我方未确认的信息，只能写成待确认建议，不能写入正式商品事实。
- 缺信息时继续规划，并列出可补充信息。
- 图片无法打开时记录具体路径并列入视觉复核清单，不得假装看懂。

## 22. 质量检查

输出前逐项检查：

- 标题是否不超过 255 字符。
- 搜索关键词是否不超过 250 字符。
- 5 点商品亮点总字符是否不超过 1500。
- 最大卖点是否贯穿标题、首图、详情图、描述、亮点和生图提示词。
- 关键词是否自然，是否与商品真实相关。
- 美国表达是否自然，是否避免中文直译和跨境模板腔。
- 主图顺序是否符合点击、理解、证明、信任和转化链路。
- 详情图是否移动端友好，是否一图一任务。
- 是否简单拼接多个竞品卖点。
- 是否混淆不同卖家的规格、参数、配件或版本。
- 是否直接复制竞品完整文案、构图、包装、人物素材或品牌化视觉。
- 是否每张主图和详情图都单独输出提示词。
- 生图提示词是否锁定产品结构、颜色、比例、数量、配件、材质和关键识别点。
- 是否添加用户和竞品都没有的信息。
- 缺失信息是否列入“可补充信息”。

## 24. 运行包维护规则

任何核心规则变更后，都必须同步检查：

1. `00_CHATGPT_READ_FIRST.md`
2. `CHATGPT_RUNTIME_BUNDLE.md`
3. `changelog.md`

必须同步运行包的情况：

- 最大卖点评分规则变化。
- 竞品产品组聚合规则变化。
- 标题或搜索关键词规则变化。
- 主图顺序或详情图逻辑变化。
- 商品描述或 5 点亮点规则变化。
- 美国本土语言规则变化。
- TikTok Shop 流量与转化逻辑变化。
- ChatGPT 生图提示词模板变化。
- 最终输出结构变化。
- 质量检查规则变化。

不进入运行包的内容：

- 新增研究来源但没有产生新规则。
- 原始竞品商品信息、竞品图片和完整竞品分析。
- URL 清单、网页摘要、视频逐条学习记录和研究过程。
- 单次任务临时测试结果。
- Git 操作记录。
- `.gitkeep` 说明。
- 只对单次任务有效的内容。

维护时只加入当前有效、稳定、可执行的规则；重复内容合并；删除冗余解释；更新来源提交号、生成日期和运行包版本；同步更新 `changelog.md`。
